Честно о барьерах

«У меня уже есть контекст в других чатах — не хочу начинать с нуля»

Знакомо? Вы уже что-то обсуждали с AI-моделью, где-то есть полезные чаты, и мысль о переносе всего контекста в новое место вызывает сопротивление. Это нормально — и вот что снимает этот барьер.

🎙 Голосовой ввод решает 80% проблемы

Вам не нужно печатать контекст. Надиктуйте голосом — это занимает 5-10 минут на проект. Инструменты вроде MacWhisper (бесплатный тариф) отлично распознают даже микс русского с англицизмами: «ROAS D3», «CPP», «screen mirroring» — всё запишется корректно на английском, остальное на русском.

🧊 Первый раз — тупишь, и это нормально

При первой надиктовке вы будете думать «а что вообще рассказать?». Лайфхак: попросите Claude дать вам базовую структуру — отправьте голосовое сообщение «я хочу скормить тебе контекст по моему проекту, дай мне чеклист что тебе нужно знать» — и дальше просто отвечайте на пункты. Со временем привыкнете и будете надиктовывать свободно.

💡 Аха-момент: бесшовный опыт без затупов

Реальный переломный момент наступает когда вы видите что Claude отдаёт файлы, верстает лендинги, считывает ваши Excel-выгрузки — всё без ошибок, с первого раза. Это как второй мозг, только быстрее.

И обратная сторона: если AI-модель хоть раз затупит при выдаче файла или считывании данных — доверие падает моментально. Появляется негативный bias, который потом мешает вообще пробовать. Поэтому выбор инструмента, который работает стабильно — критичен.

Зачем это нужно

Проблема: каждый новый чат — с чистого листа

Без настройки каждый разговор с AI начинается одинаково: «У меня есть приложение для screen mirroring, мои конкуренты — X, Y, Z, мы закупаем трафик через ASA, KPI — ROAS D3 не ниже 15%...». Через неделю это надоедает, и вы начинаете давать AI-модели меньше контекста — а значит получать менее точные ответы. В Claude эта проблема решается системой проектов.

Принцип: 1 профессиональная роль = 1 проект в Claude

Не нужно дробить отдельные приложения или каналы трафика на разные проекты Claude. Один проект «UA Specialist» покрывает все ваши рабочие приложения — потому что контекст между ними связан: подходы переносятся, конкуренты пересекаются, а метрики одни и те же. Отдельный проект — для личных дел, если хотите разделить.

Почему не дробить по приложениям?

Приложения пересекаются: удачный подход к CPP из Smart Mirror можно протестировать на EveryScan, конкурент из одного проекта может оказаться релевантным другому, правила кластеризации общие. Один проект = весь контекст доступен.

2

Задайте инструкции проекта

Инструкции проекта — это текст, который Claude видит в каждом чате внутри проекта. Сюда идут правила, которые вы хотите применять всегда, не повторяя каждый раз.

📌 Project Instructions — пример
## Как работать с семантикой При работе с семантикой и кластеризацией — если замечаешь паттерн, кластер или группировку, которую я не задала явно, всегда предлагай. Если считаешь что ключ лучше подходит в другой кластер чем тот что следует из правил — скажи почему. Если видишь локальные слова-синонимы или необычные конструкции в конкретном гео — обрати на них внимание. Не ограничивайся только тем что я попросила. ## Общие правила Если видишь проблемы в моей логике либо знаешь свежие или фундаментальные идеи, которые я не использую, но которые могут быть применимы — предлагай.
Инструкции ≠ контекст

Инструкции — это поведение AI (как он себя ведёт). Контекст — это знания (что он знает о ваших проектах). Не путайте: описание приложения идёт в контекстный файл, а не в инструкции.

3

Создайте контекстный файл на каждое приложение

Один файл = один проект/приложение. Надиктуйте голосом — не нужно структурировать идеально. Claude разберётся. Но вот шаблон-подсказка, чтобы не забыть ничего важного:

Не парьтесь с форматированием

Надиктуйте голосом — Claude понимает неструктурированный текст. Шаблон выше — подсказка чтобы ничего не забыть, а не обязательный формат. Можно просто наговорить 5 минут «thoughts dump» и загрузить как есть.

4

Организуйте чаты по задачам, а не по приложениям

Не делайте «1 чат = 1 приложение». Claude ориентируется по контексту файлов, а не по чатам. Чаты удобнее организовывать по типу задачи — так проще и вам, и Claude.

Примеры чатов по задачам

  • Weekly Reports — сюда скидываете данные в течение недели, в пятницу просите собрать отчёт. Все проекты — в одном чате.
  • Семантика Smart Mirror BR — загрузка выгрузок, кластеризация, очистка ключей для конкретного гео.
  • CPP-тесты Smart Mirror — анализ перформа CPP, сравнение метрик, выводы.
  • Конкуренты EveryScan — скриншоты листингов, анализ отзывов, преимущества и слабости.
  • Семантика Heart Rate US — отдельная задача по ключам для конкретного проекта и гео.

5

Включите «Search and reference past chats» — и используйте правильно

Это настройка, которая позволяет Claude искать по вашим прошлым чатам внутри проекта. Без неё каждый чат — изолирован. С ней — Claude может достать информацию из других чатов.

⚡ Ключевой нюанс — без этого фича не работает

«Search and reference past chats» — это не автоматическая память между чатами. Claude не «помнит всё» из прошлых разговоров по умолчанию. Он ищет по ним — но только когда вы явно ссылаетесь на прошлое.

Это значит: чтобы Claude подтянул данные из другого чата, вам нужно дать ему триггер — упомянуть прошлый контекст или задать вопрос, который подразумевает информацию из прошлых разговоров.

✓ Работает — явная ссылка на прошлое
«В прошлом отчёте мы видели ROAS D3 14% по Smart Mirror US — какая динамика на этой неделе?»
✓ Работает — вопрос подразумевающий прошлый контекст
«Какие ключи мы добавляли в samsung-кластер BR в прошлый раз?»
✓ Работает — запрос динамики
«Как менялся CPI по Heart Rate за последние 3 отчёта?»
✗ Не работает — Claude не подтянет прошлое автоматически
«Собери отчёт за неделю» (если данные были в другом чате и вы не упомянули это)
Практический вывод

Ведите weekly-отчёты в одном чате — тогда вся история метрик будет в контексте. А когда хотите подтянуть данные из другого чата — просто упомяните прошлое: «как мы обсуждали», «в чате по конкурентам мы нашли», «по данным из прошлого анализа CPP». Этого достаточно чтобы Claude нашёл нужный контекст.

Advanced Tips

Как получить ещё больше от этой системы

📈

Используйте weekly-чат как аналитический инструмент

Если вы ведёте все отчёты в одном чате, Claude накапливает историю метрик по неделям. Попросите: «подсвети проекты где ROAS D3 падает три недели подряд» или «какие победы за последний месяц можно презентовать руководству». Данные уже в контексте — Claude просто анализирует их.

🔗

Подключите Slack как коннектор

Если ваши рабочие обсуждения, статусы и решения живут в Slack — подключите его. Claude сможет считать ваши сообщения за неделю и автоматически собрать список выполненных задач для отчёта, без ручного вспоминания «а что я делал во вторник».

🔄

Переносите подходы между приложениями

Один проект Claude для всех приложений — это не только удобство, но и стратегия. Если CPP с фокусом на бренды ТВ сработала на Smart Mirror — попросите Claude адаптировать подход для другого приложения. Весь контекст обоих проектов уже на месте.

📎

Обновляйте контекстные файлы по мере изменений

Сменился KPI, добавился новый канал закупки, нашли нового конкурента — обновите файл. Это занимает 2 минуты, а Claude сразу начнёт учитывать обновлённый контекст во всех новых чатах. Не нужно «переобучать» — просто поправьте файл.

🌐

Ссылка на приложение — не просто для справки

Если в контекстном файле есть ссылка на App Store / сайт, Claude может зайти и посмотреть актуальный листинг: текущие скриншоты, описание, отзывы. Полезно когда вы обсуждаете CPP-тесты или конкурентный анализ — Claude видит живую версию, а не ваше описание двухмесячной давности.
